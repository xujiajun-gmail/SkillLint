# 内容引擎 (Content Engine)

将一个想法转化为强大的、平台原生（platform-native）的内容，而不是在所有地方发布同样的东西。

## 何时激活 (When to Activate)

- 编写 X 帖子或推文串（threads）
- 起草 LinkedIn 帖子或发布更新
- 编写短视频（short-form video）或 YouTube 讲解视频脚本
- 将文章、播客（podcasts）、演示（demos）或文档重用为社交内容
- 围绕发布、里程碑或主题构建轻量级内容计划

## 首要问题 (First Questions)

澄清以下内容：
- 来源资产（source asset）：我们要从什么内容进行适配
- 受众（audience）：开发者（builders）、投资者、客户、运营者或普通受众
- 平台（platform）：X、LinkedIn、TikTok、YouTube、时事通讯（newsletter）或多平台
- 目标（goal）：知名度、转化、招聘、权威性、发布支持或互动（engagement）

## 核心规则 (Core Rules)

1. 针对平台进行适配。不要跨平台发布相同的文案。
2. 钩子（Hooks）比摘要更重要。
3. 每条帖子应承载一个清晰的想法。
4. 使用具体的细节而非口号。
5. 保持行动号召（ask/CTA）小而清晰。

## 平台指南 (Platform Guidance)

### X
- 快速切入
- 每条帖子或推文串（thread）中的每条推文只包含一个想法
- 除非必要，否则不要在正文中放入链接
- 避免标签（hashtag）堆砌

### LinkedIn
- 强有力的第一行
- 短段落
- 围绕经验教训、结果和收获进行更明确的框架构建

### TikTok / 短视频 (Short Video)
- 前 3 秒必须抓住注意力
- 围绕视觉效果编写脚本，而不只是旁白
- 一个演示（demo），一个主张，一个行动号召（CTA）

### YouTube
- 尽早展示结果
- 按章节（chapter）组织结构
- 每 20-30 秒刷新一次视觉效果

### 时事通讯 (Newsletter)
- 提供一个清晰的视角，而不是一堆无关项的集合
- 使各节标题易于扫描（skimmable）
- 让开头段落发挥实际作用

## 重用流程 (Repurposing Flow)

默认级联步骤：
1. 锚点资产（anchor asset）：文章、视频、演示、备忘录（memo）或发布文档
2. 提取 3-7 个原子创意（atomic ideas）
3. 编写平台原生（platform-native）变体
4. 削减输出中的重复内容
5. 使行动号召（CTAs）与平台意图保持一致

## 交付物 (Deliverables)

当被要求策划活动（campaign）时，返回：
- 核心视角
- 平台特定的草案
- 可选的发布顺序
- 可选的行动号召（CTA）变体
- 发布前所需的任何缺失输入

## 质量关卡 (Quality Gate)

在交付前：
- 每个草案读起来都符合其平台的原生风格
- 钩子（hooks）强有力且具体
- 没有通用的炒作语言
- 除非要求，否则不同平台之间没有重复文案
- 行动号召（CTA）与内容和受众匹配
