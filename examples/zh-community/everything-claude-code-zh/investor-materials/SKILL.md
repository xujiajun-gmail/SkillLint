# 融资材料（Investor Materials）

构建一致、可靠且易于辩护的面向投资者的材料。

## 何时激活（When to Activate）

- 创建或修改路演 PPT（Pitch Deck）
- 编写投资备忘录（Investor Memo）或一页纸简介（One-pager）
- 构建财务模型、里程碑计划或资金用途表（Use-of-funds table）
- 回答加速器（Accelerator）或孵化器申请问题
- 围绕单一事实来源（Source of Truth）对齐多个融资文档

## 黄金法则（Golden Rule）

所有融资材料必须保持一致。

在编写之前，创建或确认一个单一事实来源（Single Source of Truth）：
- 增长/牵引力指标（Traction Metrics）
- 定价和收入假设
- 融资规模和融资工具
- 资金用途
- 团队简历和职位
- 里程碑和时间表

如果出现矛盾的数字，请在起草前停止并解决它们。

## 核心工作流（Core Workflow）

1. 盘点权威事实
2. 识别缺失的假设
3. 选择资产类型
4. 带着明确的逻辑起草资产
5. 对照事实来源交叉检查每一个数字

## 资产指南（Asset Guidance）

### 路演 PPT（Pitch Deck）
推荐流程：
1. 公司 + 切入点（Wedge）
2. 痛点（Problem）
3. 解决方案（Solution）
4. 产品 / 演示（Demo）
5. 市场（Market）
6. 商业模式
7. 增长/牵引力指标（Traction）
8. 团队
9. 竞争 / 差异化
10. 诉求（Ask）
11. 资金用途 / 里程碑
12. 附录

如果用户想要一个 Web 原生的幻灯片，请将此技能与 `frontend-slides` 配合使用。

### 一页纸简介 / 备忘录（One-Pager / Memo）
- 用一句精炼的话说明公司是做什么的
- 展示“为什么是现在”
- 尽早列出增长和证明点
- 使诉求（Ask）精确
- 确保声明易于验证

### 财务模型（Financial Model）
包含：
- 明确的假设
- 在有用时提供悲观（Bear）/ 基础（Base）/ 乐观（Bull）情况
- 清晰的逐层收入逻辑
- 与里程碑挂钩的支出
- 决策取决于假设时的敏感性分析

### 加速器申请（Accelerator Applications）
- 回答所提的每一个具体问题
- 优先考虑增长指标、行业见解和团队优势
- 避免吹嘘
- 保持内部指标与 PPT 和模型一致

## 需避免的红线（Red Flags to Avoid）

- 无法核实的声明
- 缺乏假设的模糊市场规模
- 不一致的团队角色或职衔
- 无法清晰求和的收入计算
- 在假设非常脆弱的情况下表现出过度的确定性

## 质量把关（Quality Gate）

在交付之前：
- 每一个数字都与当前的事实来源相符
- 资金用途和收入层级求和正确
- 假设是可见的，而不是被掩盖的
- 叙述清晰，不使用夸张语言
- 最终资产在合伙人会议上是经得起推敲的
