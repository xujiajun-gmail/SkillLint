# 项目整体梳理

按用户提供的「项目梳理格式」或文档结构，基于当前项目代码与仓库信息生成项目整体梳理内容。

## 使用时机

- 用户提供一份「项目梳理」的格式、大纲或模板（如「一、项目概述 二、模块划分 三、技术选型 四、目录结构 五、核心接口」）
- 用户提到「项目整体梳理」「按这个格式梳理我们项目」「生成项目文档结构」
- 需要将代码仓库整理成结构化的项目说明（课程作业、毕设、内部文档）

## 总体思路

1. **解析用户格式**：识别一级/二级标题、表格、每部分期望深度
2. **从当前项目抽取**：README、目录、配置、入口、路由/API 等
3. **按格式组织**：严格按用户标题顺序填写，缺失部分标注「暂无」或「需补充」

## 工作流程

1. **读取并解析梳理格式**：大纲或示例文档，提取结构、表格列名、粒度
2. **从仓库抽取**：项目概述、功能模块、技术选型、目录说明、核心接口、运行方式等
3. **按格式生成**：遵循标题层级与顺序，表格按实际数量填写，不虚构
4. **输出与补充建议**：完整正文，未填项注明「需补充」

## 注意事项

- 不编造：未在仓库出现的模块、接口、技术不写入
- 结构以用户格式为准：不随意增加用户未要求的大节
