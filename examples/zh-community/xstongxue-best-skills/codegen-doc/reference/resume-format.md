# 简历项目描述

按用户提供的简历/项目经历格式，基于当前项目代码与仓库信息生成对应的「项目描述」或「经历条目」。

## 使用时机

- 用户提供简历格式示例或模板（如「项目名称/时间/技术栈/描述/职责/成果」），希望按格式为当前项目生成「项目经历」
- 用户提到「按简历格式写这个项目」「把当前项目写成简历里的项目描述」
- 需要将 GitHub 项目、课程项目、毕设项目整理成简历可用的表述

## 总体思路

1. **解析用户格式**：识别字段（项目名、时间、角色、技术栈、描述、职责、成果、亮点）及写法、长度、语气
2. **从当前项目抽取**：README、package.json、目录结构、关键代码中的项目名、技术栈、功能点、可量化信息
3. **按格式填充并润色**：将抽取内容填入格式，保持与示例一致风格与长度

## 工作流程

1. **读取并解析格式**：从示例/模板识别固定字段、写作风格（动词开头、分号分隔、是否带数据）
2. **从仓库抽取**：项目名称与简介、技术栈、功能与模块、可量化信息（接口数、模块数等，仅在有依据时写）
3. **按格式生成**：严格按字段顺序、分段方式、标点习惯；占位符（时间、公司、角色）标注「请自行替换」
4. **输出与占位说明**：完整一条或一段，注明需用户替换的占位项

## 注意事项

- 不虚构：未在仓库或 README 中出现的技术、功能、数据不写入
- 不夸大：避免「主导」「独立完成整个系统」；优先用「参与」「负责」「实现」等可证实表述
