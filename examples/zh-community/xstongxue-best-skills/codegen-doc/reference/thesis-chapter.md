# 硕士论文章节生成（系统设计向）

根据用户提供的章节大纲和当前项目代码/需求文档，自动撰写整章内容（含系统架构与系统流程），输出适配 Word 的中文学术文风文本。

## 使用时机

- 用户希望「根据章节大纲生成硕士论文章节内容」
- 用户提到「系统设计」「总体设计」「详细设计」「系统实现」「系统架构图」「系统流程图」
- 需要围绕当前项目生成结构化的论文文字说明

## 总体原则

- **面向具体项目**：从代码结构、模块划分、配置文件、文档中抽取信息，不凭空泛写
- **中文学术风格**：客观、克制，多用「本课题」「本系统」「本章」，避免「我们」
- **去 AI 化**：避免「显著提升」「极大增强」「卓越表现」等 AIGC 痕迹词汇
- **Word 适配**：禁止 Markdown、全角中文标点、中英文间加空格

## 工作流程

1. **明确输入**：章节名称、大纲、项目背景、字数范围
2. **收集项目上下文**：README、入口、配置、部署相关文件
3. **抽取系统架构**：子系统/模块、职责、部署形态、关键组件
4. **设计系统流程**：1–3 个典型流程（用户操作、数据处理、服务调用）
5. **组织章节**：总-分-总，章节引言、系统概述、总体设计、流程设计、关键技术、本章小结
6. **输出格式**：无 Markdown，普通段落，可分段首用「（一）」「1）」等序号

## 结构要点

- 系统架构：给出「图 x.x 系统总体架构图」的文字说明及分层说明
- 系统流程：流程概述 + 步骤描述 + 可选「节点-连线」清单供绘图
- 关键技术与实现：选取与项目强相关的技术点，适度提及类/模块名
