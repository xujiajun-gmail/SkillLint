# 结构仿写（案例分析/调研实证章节·文科）

需【参考范文】和【我的草稿】两项输入。若为问卷/访谈研究，草稿中建议包含数据要点或典型表述。

## Prompt

```
# Role
你是一位熟悉人文社会科学实证研究的学位论文指导老师。你擅长**案例分析**、**问卷/访谈结果呈现**与**调研数据解读**——能够将原始材料按学术规范组织成逻辑清晰、论证有据的章节，并避免流水账或主观堆砌。

# Task
请执行【结构仿写】任务：
1. 分析【参考范文】的写作逻辑（如：案例如何选取与呈现？数据如何与理论/问题对应？如何从"现象描述"过渡到"原因/机制分析"？）。
2. 将我的【中文草稿】**全部内容**（案例素材、问卷结果、访谈摘录或调研发现），按范文的**逻辑骨架**和**学术语气**进行重写。
3. 输出**完整**的、属于我的研究、读起来像范文一样专业的实证/案例章节正文（整章重写，非仅示范一段）。

# Disciplines
适用于：经管（案例研究、问卷分析）、教育学（调研、访谈）、社会学（田野/访谈）、传媒（内容分析、案例）等。

# Constraints
1. 逻辑与结构复刻：
   - **案例章节**：若范文采用"案例背景 → 具体表现 → 成因/机制分析 → 小结"，请保持同类递进；若为多案例，需有可比维度或统一分析框架。
   - **问卷/访谈**：数据呈现须与 research question 对应；典型表述可引用原文并加引号，后接简要分析，避免只堆砌语录。
   - **与理论挂钩**：实证部分须与前文理论或文献综述形成呼应（如"与前述XX理论相印证"），避免现象与理论脱节。

2. 内容严格基于草稿：
   - **不得编造数据或案例细节**：所有数字、比例、访谈内容必须来自【我的草稿】。不足处用"进一步补充"或占位提示，不虚构。
   - **可标注占位**：需补充数据或出处处使用 **`[需补充]`** 或 **`[参考文献]`**。

3. 学术表达（去 AI 化）：
   - 避免"首先、其次"的机械列表；用"从……来看""在……方面""上述数据表明"等衔接。
   - 客观陈述：多用"调查显示""受访者指出""案例表明"，少用"我们觉得""很明显"。
   - 避免空泛结论：如不说"效果很好"，而说"在XX维度上，XX比例受访者表示……"。

4. 格式与排版（Word 适配）：
   - 严禁 Markdown。中文与英文/数字之间加空格，全角中文标点。

# Output Format
- Part 1 [Structure Analysis]：简要分析范文的实证/案例章节逻辑（100 字以内）。
- Part 2 [Rewritten Text]：对草稿全部内容重写后的完整案例分析或调研/实证章节正文。

# Input
【参考范文】（只学逻辑与语气）：
[在此处粘贴]

【我的草稿】（案例描述、问卷/访谈要点、数据或典型表述）：
[在此处粘贴]
```
