# 个人实测 / 软件推荐写作风格

参考「软件推荐」「工具盘点」等高流量个人实测类文章。当用户要求「个人实测」「亲身推荐」时使用本指南。

## 核心原则

- **个人经历驱动**：用真实经历（健康问题、踩坑、需求触发）作为开头，增强共鸣
- **坦诚实测**：推荐旧版不推荐新版、推荐某渠道不推荐官网，说明具体原因
- **背景科普**：在推荐前简要科普原理/依据（如防蓝光原理、医生建议、行业标准）
- **实用落地**：每个推荐都有官网、下载方式、操作技巧

## 开头结构

1. **个人经历**：最近 XX 出了点问题，去做了详细检查，这次才真正下定决心…
2. **过往努力**：尽管我早在两年前就已经做了很多（列举 2–4 项具体措施及依据）
3. **转折**：但可能这些做的都还是不足够…
4. **背景科普**：XX 这个词相信大家都不陌生（简要原理、研究/医生建议）
5. **个人痛点**：我自己经常会…再加之…所以各种问题都会更容易出现
6. **引出主题**：所以最近又加上了 XX，这次也顺便来给大家推荐一下

## 背景科普（可选但加分）

- 用 1–2 段简要说明原理、标准、建议
- 示例：防蓝光（蓝光危害、偏暖色温）、医生建议（短暂频繁休息）、行业标准（环境光与工作区亮度比 1:3）
- 增强可信度，让读者理解「为什么需要」

## 推荐结构

- 按数字编号：1、2、3… 每项一个软件/工具
- 每项 5–8 个子项，按推荐优先级排序
- 可标注「阿虚自己选择用的是 XX」突出个人偏好

## 单项推荐格式

每个软件/工具包含：

1. **标题**：软件名
2. **官网链接**：放在开头
3. **一句话介绍**：谁家的、特点（免费/付费、国内外出名等）
4. **核心功能**：4 种模式、定时提醒、冷暖色调节等
5. **操作技巧**：坐标填写、百度地图拾取、需手动辅助调整等
6. **版本/渠道建议**：推荐旧版不推荐新版、推荐某渠道不推荐官网，说明原因（新版阉割了 XX、旧版可自定义 XX）
7. **缺点**：只有英文界面、延迟时间固定、功能欠缺等
8. **下载方式**：官网 / 储物间 X–X 栏 / 搜索关键词

## 排除说明（文末）

- 在正文推荐后，用一段话列出**不推荐**的软件及原因
- 格式：软件 A：有 XX 功能，但缺点是不能…；软件 B：多种模式，但休息间隔不能自定义…
- 收束：为了文章不废话，优中取优，上面省略了很多不推荐的
- 可加「↕上下滑动查看更多↕」等提示

## 结尾结构

1. **健康/行动提醒**：无论软件能起到多大的辅助，最关键的都是要 XX＋XX
2. **情感收束**：大家可要好好保护 XX 啊

## 语气与用词

- **第一人称**：阿虚、我、我自己
- **坦诚**：可能这些做的都还是不足够、阿虚自己选择用的是、阿虚认为非常重要的
- **实用导向**：好在软件还内置、最关键是可以、缺点就是

## 篇幅与排版

- 建议 4000–7000 字，长文可读性强
- 可穿插配图（软件界面、设置截图）
- 重要操作步骤可单独成段

## 适用场景

- 健康/护眼类软件推荐
- 个人实测的工具盘点（有真实使用经历）
- 需要背景科普 + 差异化建议的软件合集
